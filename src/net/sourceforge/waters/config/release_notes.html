New in version 2.4

  • Normalising compiler semantics. The default semantics for new installations is the normalising semantics, which keeps variables unchanged unless explicitly assigned on a transition. You can change the compiler semantics in the Options, under the gui tab, by checking or unchecking: Use normalising EFSM compiler.
  • Global font and display scaling. It is possible to enlarge or shrink the whole IDE including all icons and fonts by changing the Icon set under the gui.editor tab in the Options.
  • Zoom. The size at which automata are displayed can be changed using the Zoom box in the editor toolbar.
