New in version 2.4.1

  • Automaton variables. It is now possible to reference the names of arbitrary automata in guard/action blocks. For example, if the model includes a plant mach1 with a state idle, then a guard mach1==idle in any automaton restricts the events of its transition to situations where plant mach1 is in state idle. To use this feature, both Use normalising EFSM compiler and Allow automaton names in EFSM guards must be checked in the gui tab in the Options. Enabling automaton variables introduces additional syntax restrictions, as there cannot be clashes between variables, automata, state names, and enumeration constants.

    • New in version 2.4

    • Global font and display scaling. It is possible to enlarge all icons and fonts in the IDE by changing the Icon set under the gui.editor tab in the Options. This setting will take effect the next time you start the IDE.
    • Zoom. The size at which automata are displayed can be changed using the Zoom box in the editor toolbar.
