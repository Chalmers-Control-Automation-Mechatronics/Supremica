central_locking
Description Central Locking example. A large model of a car's central locking system, which originally was developed as a case study within the BMBF project KORSYS, and at that time was considered as a challenge for formal verification.

Demonstrated		 Powerful incremental verification

How to Use
As a comparatively large model, the central locking system can be used to demonstrate the verification powers of DESign. The model verriegel3, e.g., consists of 54 automata, and has more than 750,000,000 reachable states. Using DESign, the following checks can be performed:
Controllability
All models are controllable, with verification times of a few seconds.
Conflict
All models are nonconflicting, with verification times of a few minutes. In order to see an interesting conflicting example, it is possible to delete the template called noreset from project verriegel3 or verriegel4. The counter example should be computed in a few seconds.
Language Inclusion
Two properties are included for language inclusion checking.
  • The automaton sicherheit_vr3 represents the property that the car's rear doors are only locked if a locking command from one of the keyholes has occurred before. This automaton can be added to project verriegel3, and the language inclusion check showing that the property is satisfied takes a few seconds. Similarly, it is possible to add the automaton sicherheit_vr4 to project verriegel4 in order to perform the check on the larger model.
  • The automaton sicherheit_er represents the property that the car's driver door is only unlocked if an unlocking command from one of the keyholes has occurred before. This property is not satisfied in the model. The automaton can be added to project verriegel3 or verriegel4, and a language inclusion check of a few seconds will produce a counter example.
