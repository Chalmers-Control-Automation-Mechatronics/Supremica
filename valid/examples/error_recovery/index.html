modular_recovery
A small workcell consisting of a coordinator and two components, with the possibility of an error occurring at any time.
How to Use Open project modular_recovery and look at its modular structure.
The plant model error states that an error event may occur at any time, but the reaction to this error depends on whether one of the two components comp1 and comp2 is running or not. If both components are idle, then a RepairInit should occur after an error, if comp1 is running then it should be Repair1, and if comp2 is running then it should be Repair2. Now, the specification recovery only states that every error should be followed by some recovery; the actual selection of the recovery procedure is performed by the automata comp1 and comp2, which cooperatively enable one of the three events.
