conflict
Description The classical coordination problem of two users competing for access to the same two resources.
Features
Demonstrated		 Conflict checking
Projects
deadlock
A conflicting project, representing the uncoordinated access of the two users to the resources.
fixed
A modified version of the deadlock project, which has been fixed by adding a coordinator unit.
How to Use Open project deadlock and see that it is conflicting. The error trace obtained from the conflict check shows how the two users run into deadlock, if user 1 grabs resource A and user 2 grabs resource B before user 1 can get hold of it. Project fixed shows how this problem can be fixed by adding the automaton coordinator as an arbitration unit.
