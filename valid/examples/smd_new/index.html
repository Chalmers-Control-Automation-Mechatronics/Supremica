smd_new
A new concurrent version of the scan-vacuum test for the twelve pipettes, which fixes the deadlock in the example smd_concurrent. The automata scantask and vacuumtask now have very similar structure, all activities related to error recovery have been moved to a new specification called recoverytask.
How to Use Open the project smd_new and look at the modelling. Also compare with the original versions smd_original and smd_concurrent.
